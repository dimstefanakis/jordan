# Core User Journey

## Flow Goal

Describe the primary end-to-end journey a user takes through the system.

## Steps

1. User enters the system.
2. User completes the first meaningful action.
3. User reaches the primary value moment.
4. User returns or escalates to support when needed.

## Branches And Edge Cases

- blocked access
- incomplete setup
- ambiguous state transitions
- handoff to a human operator

## Support-Relevant Truths

- Document only verified steps.
- If the exact UI path is uncertain, record the behavior, not the guessed navigation.
