# Company Graph

This folder is the canonical knowledge base for Jordan.

Use it to document durable knowledge about your product, workflows, operations, and support rules without depending on scattered chat memory.

## Read Order

1. `product/overview.md`
2. `flows/` for user journeys
3. `concepts/` for reusable definitions
4. `operations/` for internal procedures
5. `support/triage-policy.md`
6. `support/playbooks/` for issue-specific guidance
7. `graph/` for machine-readable routing and relationships

## Ground Rules

- Prefer documented facts over memory or guesses.
- Do not invent UI labels, navigation paths, or policies.
- Put canonical explanations here, not in ad hoc notes.
- Put temporary incidents and debugging details elsewhere.

## Workbench

Use `groups/main/knowledge/` for raw notes, imported docs, and draft synthesis before normalizing them into this graph.
