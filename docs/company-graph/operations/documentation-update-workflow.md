# Documentation Update Workflow

## When To Use This

Use this procedure when durable knowledge needs to be added or corrected in the company graph.

## Prerequisites

- a concrete fact, decision, or workflow update
- enough context to place it in the correct family

## Steps

1. Find the current chapter that should own the knowledge.
2. If no chapter fits, create a new one using `_conventions.md`.
3. Update graph metadata if chapter structure changed.
4. Keep the edit factual and minimal.

## What To Verify After

- the canonical chapter exists
- the graph metadata references it when needed
- duplicated explanations are removed or linked instead
