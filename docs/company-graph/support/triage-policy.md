# Support Triage Policy

## Purpose

Use this document to classify incoming requests before responding.

## Default Rules

- Prefer guidance grounded in the company graph.
- If the request would mutate data, change account state, or take an external action, classify it as human-required unless a later operations document says otherwise.
- If details are missing, ask for clarification instead of guessing.
- If the issue appears to reveal a documentation gap, ask Atlas to update the company graph after handling the request.

## Common Classifications

- `info-only`
- `ask-more-info`
- `human-check-required`
- `human-action-required`
- `human-escalation-required`
